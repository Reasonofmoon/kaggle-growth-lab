# Rules Summary

This is a practical summary. Check the official Kaggle rules before submitting.

## Core Facts

| Item | Value |
|---|---|
| Competition | Neural Debris Removal in Streak Detection Models |
| Sponsors | European Space Agency and Sybilla Technologies |
| Competition website | https://assurance-ai.space-codev.org/competitions/ |
| Data license | CC BY 3.0 IGO |
| Final submission deadline | 2026-07-22 Anywhere on Earth |
| Submission limit | 2 per 24 hours per team |
| Max team size | 3 |

## Eligibility

The competition is open broadly, including space experts, data-mining experts, and citizen scientists.

Monetary prizes are restricted to nationals of listed eligible countries. South Korea and the United States are included in the pasted rules.

## Prize Verification

Prize-eligible winners must share solution details before 2026-08-07 so organizers can verify rule compliance.

This may include:

- code
- short solution description
- enough detail to run and verify the method

## Code Sharing

During the competition:

- private sharing between different teams is not allowed
- public sharing is allowed through Kaggle competition forums/kernels
- public code sharing must not violate third-party IP
- open-source code must use OSI-approved licenses and allow commercial use

## Test Set Rule

The most important rule:

```text
The test set must not be annotated in any way.
```

This includes:

- manual labels
- hard labels
- weak labels
- soft labels
- pseudo-labels

Predictions generated by the provided poisoned model are not considered annotations under the pasted rules.

Unlabeled test images may be used as reference data in de-poisoning, but this should be handled conservatively and documented carefully.

## Data Security

Do not redistribute competition data to people who have not accepted the competition rules.

Do not commit raw competition data or model weights to GitHub.
