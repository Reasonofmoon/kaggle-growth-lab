# Competition: ROGII - Wellbore Geology Prediction

Kaggle: https://www.kaggle.com/competitions/rogii-wellbore-geology-prediction

This is a notebook-only regression competition for predicting `tvt` along horizontal wellbores.

## 1. Goal

Build a model that predicts True Vertical Thickness (`tvt`) for evaluation zones in horizontal wells.

My goal for this competition is to practice a clean ML workflow:

- dataset inspection
- EDA
- well-aware validation
- simple baseline
- feature engineering
- model comparison
- reproducible notebook submission

## 2. Why This Fits Kaggle Growth Lab

This is the most standard machine learning competition in the current project.

Compared with the agent/simulation competitions, ROGII is better for demonstrating:

- regression modeling
- leakage-aware validation
- feature handling
- notebook documentation
- metric-driven iteration

## 3. Metric

Submissions are scored using RMSE.

Lower is better.

## 7. First Baseline Scope

The first notebook should:

1. List train/test wells.
2. Inspect one horizontal well file.
3. Inspect one typewell file.
4. Identify rows where `TVT_input` is missing.
5. Use nearby non-missing `TVT_input` interpolation as a baseline where possible.
6. Fall back to per-well or global median.
7. Generate `submission.csv`.

The first goal is a valid submission, not a strong score.

## 8. Validation Warning

Avoid naive random row splits.

Rows from the same well are highly correlated. A random row split can leak well-specific structure and overstate model quality.

Prefer:

- well-level validation
- group-based validation by well id
- evaluation-zone-aware checks
