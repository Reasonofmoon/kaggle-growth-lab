# I'm turning my Kaggle journey into a public engineering log

I decided to grow on Kaggle, but I do not want this to become a private learning journey.

So I am turning it into a public engineering log.

Kaggle will be my experiment field.
GitHub will be my evidence archive.
LinkedIn will be where I share the insights.

Each week, I will document:

- what problem I worked on
- what baseline I built
- what failed
- what improved
- what I learned as a developer

My goal is not only to climb Kaggle tiers, but to build a visible record of how I approach machine learning problems.

This experiment is part of my Kaggle Growth Lab.

In this week's GitHub log, I documented:

- repository setup
- current Kaggle status
- next badge targets
- first-week execution plan
